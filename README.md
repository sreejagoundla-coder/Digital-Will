# 🔐 Digital Will

Digital Will is a web-based application designed to securely manage digital assets and ensure controlled access during emergencies or periods of user inactivity. The platform focuses on maintaining digital continuity by providing a structured and responsible way to handle sensitive digital information.

---

## 📌 Overview

As the world becomes increasingly digital, individuals store critical information such as emails, cloud data, financial accounts, passwords, and personal documents online. However, there is no structured system to manage or transfer this data when a user becomes inactive or unable to access it.

Digital Will provides a secure and organized solution to manage digital assets and ensure they are accessible to trusted individuals only under predefined conditions.

---

## 📌 Problem Statement

In modern digital life, access to important data is completely dependent on the user. In cases of emergencies, accidents, or prolonged inactivity, this information becomes inaccessible to family members or trusted contacts.

This can result in data loss, financial issues, and emotional stress. Existing solutions are either insecure, unstructured, or lack proper verification mechanisms.

---

## 💡 Solution

Digital Will introduces a controlled system where users can securely store their digital assets, assign trustees, and define conditions such as inactivity triggers.

Access is granted only after proper verification and authorization, ensuring both security and trust. The system bridges the gap between digital ownership and real-world emergencies.

---

## 🚀 Key Features

- User and Trustee role-based access  
- Secure digital asset upload with file support  
- Simulated encryption for data protection  
- Inactivity trigger mechanism  
- Emergency verification via certificate upload  
- Status tracking system (Active → Under Verification → Access Granted)  
- Simulated biometric authentication (Face and Fingerprint)  
- Cloud document synchronization (prototype)  
- Multilingual support (English, Telugu, Hindi)  

---

## ⚙️ System Workflow

1. User logs into the system  
2. Uploads digital assets  
3. Assigns trusted trustees  
4. Sets inactivity triggers  
5. System detects inactivity conditions  
6. Trustee submits verification proof  
7. System verifies and updates status  
8. Secure access is granted
